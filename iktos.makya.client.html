async push_molecules(molecules)[source]¶

Pushes molecules into a generator. This method can be useful to visualise external molecules in Makya and compare them with generated molecules.

Parameters:

molecules (List[Generation]) – List of Generation objects that associates molecules with scores.

Examples

>>> from iktos.makya.client.generation.generation import MakyaMolecule, Generation, GenerationScore, CustomGenerationScore
>>> from rdkit.Chem.QED import qed
>>> from rdkit import Chem
>>> smiles = 'CCCCC'
>>> mol = Chem.MolFromSmiles(smiles)
>>> molecule = MakyaMolecule(
...     smiles=smiles,
...     inchi=Chem.MolToInchi(mol, options="-Snon -FixedH"),
...     inchi_key=Chem.InchiToInchiKey(inchi),
... )
>>> score = GenerationScore(
...     qed=qed(mol),
...     confidence=0.98,
...     custom_scores=[
...         CustomGenerationScore(score_name='IC50', value=32),
...         CustomGenerationScore(score_name='Volume', value=270),
...     ]
... )
>>> generations = [Generation(
...     molecule=molecule,
...     score=score)
... ]
>>> result = await my_generator.push_molecules(generations)

class MakyaPostProcessor(client, project, graphql_id, name, slug, scores, created_at=None)[source]¶

Bases: object

This class represents the post processor scores found in Makya Such a scorer can be used to aggregate multiple API, automl or descriptor targets and build a scoring function. The scoring function can then be used with generators to score generated molecules without impacting the reward function of the generation itself (hence the post processing part). They can also be used adhoc to compute scores for a dataset of molecules.

Be aware that the scoring itself occurs in the background. So when you submit a dataset to score or run a generation the scores may take some time to be registered in you data, so you might need to refresh the resulting dataset a few times

Parameters:

  • client (Any) – Makya client. Not user specified

  • project (Any) – Makya project. Not user specified

  • graphql_id (str) – GraphQL ID. Set by default. Not user specified

  • name (str) – Name of the Makya post-processor

  • slug (str) – GraphQL slug. Not user specified

  • scores (List[PostProcessorScore]) – List of scores that are aggregated in the scorer

  • created_at (Optional[str]) – Creation date of the predictor set. Not user specified

Example

>>> from iktos.makya.client.post_processor_configuration import (
...     PostProcessorApiScore,
...     PostProcessorAutoMLScore,
...     PostProcessorDescriptorScore
... )

>>> # Print supported descriptors from RDKit
>>> from iktos.makya.client.post_processor_configuration import valid_descriptor_names
>>> valid_descriptor_names

>>> # RDKit descriptor
>>> score1 = PostProcessorDescriptorScore(descriptor="NumRings")
>>> # API Scorer
>>> external_scorers = await project.get_scorers()
>>> score2 = PostProcessorApiScore(scorer=external_scorers['my API scorer'])
>>> # AutoML predictor
>>> predictor_sets = await project.get_predictor_sets()
>>> predictor_set = predictor_sets['my predictor set']
>>> predictors = await predictor_set.get_predictors()
>>> score3 = PostProcessorAutoMLScore(predictor=predictors["Target1"])
>>> # Creates the post-processing scorer
>>> new_scorer = await project.create_post_processor(
...     name="my scorer",
...     scores=[
...         score1,
...         score2,
...         score3
...     ]
... )

async show_all_metrics()[source]¶

Displays the metrics allowing the user to evaluate the performances of the predictor set. The metrics are displayed as an ipywidgets with one tab per objective of the model. In each tab, the following performance indicators are shown for both the train and test sets:

  • Precision: a measure of how many of the positive predictions made are correct (truly positive cases).

  • Recall: a measure of how many of the positive cases were correctly predicted by the classifier.

  • AUC: Area Under the Curve of the ROC (Receiver Operating Characteristic) curve, a measure of the ability of classifier to distinguish between classes.

  • F1-score: a score that combines precision and recall in a balanced way.

  • ROC curve: the basis for the calculation of the AUC. A plot of the False Positive Rate vs True Positive Rates at various thresholds.

  • Confusion matrix: a table that reports the number of true positives, true negatives, false positives and false negatives.
